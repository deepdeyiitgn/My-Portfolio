## 🐛 Before Opening a Bug Report

Please check the following before submitting a new issue:

1. **Confirm it's reproducible** — try a hard refresh (Ctrl+Shift+R / Cmd+Shift+R) and an incognito window.
2. **Check existing issues** — search [open issues](https://github.com/deepdeyiitgn/My-Portfolio/issues) to avoid duplicates.
3. **Check the latest branch** — your issue may already be fixed in a recent commit.

**Required in your report:**
- Steps to reproduce (numbered, exact)
- Expected behavior
- Actual behavior
- Browser, OS, and device type
- Screenshots or screen recording if the issue is visual

---

## 💡 Before Opening a Feature Request

Feature requests are welcome but given the current academic hiatus, review may be delayed. Before submitting:

1. **Check if it already exists** — browse the site at [deepdey.vercel.app](https://deepdey.vercel.app) and search existing issues.
2. **Describe the value clearly** — what problem does this solve and for whom?
3. **Keep scope small** — incremental improvements are easier to review than complete redesigns.

## 🔒 Security Issues

**Never open a public issue for a security vulnerability.**

Follow the private reporting process documented in [SECURITY.md](./SECURITY.md):
- Email: **a@qlynk.me** or **team.deepdey@gmail.com**
- Subject: `[SECURITY REPORT] <description>`
